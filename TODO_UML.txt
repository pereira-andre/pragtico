## 2. Diagrama de Sequência: chat web -> resposta do bot

### Prioridade

Alta

### Porque é necessário

Explica o fluxo principal do bot no canal web e mostra onde entram:

- parsing de comandos
- pending actions
- RAG
- persistência
- feedback

### Fluxo a representar

1. utilizador envia mensagem no portal
2. `blueprints/chat.py` recebe o pedido
3. `handle_chat_turn(...)` orquestra o turno
4. consulta histórico e pending action em `storage`
5. tenta resposta determinística/comando
6. se necessário chama `rag.answer(...)`
7. persiste mensagens
8. devolve resposta ao frontend

### Elementos a destacar

- ramo `slash command`
- ramo `consulta normal`
- ramo `pending action`

## 3. Diagrama de Sequência: inbound WhatsApp -> resposta -> status

### Prioridade

Muito alta

### Porque é necessário

Este é provavelmente o diagrama que mais falta no contexto atual do projeto.

Mostra:

- webhook Meta
- normalização do evento
- ligação ao runtime comum
- envio da resposta pela Cloud API
- receção posterior de estados `sent/delivered/read/failed`

### Fluxo a representar

1. utilizador envia mensagem para o número WhatsApp
2. Meta chama `POST /webhooks/whatsapp`
3. `blueprints/whatsapp.py` faz parse e deduplicação
4. cria ou resolve utilizador interno
5. chama `handle_chat_turn(...)` com `channel="whatsapp"`
6. `integrations/whatsapp_cloud.py` envia resposta
7. `storage` guarda mensagens e eventos do canal
8. Meta envia `statuses`
9. sistema atualiza metadata da mensagem

### Elementos a destacar

- whitelist de números
- bloqueio de mutações no WhatsApp
- reações `👍/👎` como feedback

## 4. Diagrama de Estados: ciclo de vida de uma ação pendente do bot

### Prioridade

Alta

### Porque é necessário

É a forma mais clara de explicar o comportamento mais “inteligente” do bot sem ter de mostrar muito código.

### Estados a representar

- `Sem ação pendente`
- `Proposta criada`
- `À espera de campos em falta`
- `Pronta para confirmação`
- `Confirmada e executada`
- `Cancelada`
- `Bloqueada`
- `Erro na execução`

### Transições importantes

- mensagem operacional reconhecida
- pedido incompleto
- utilizador fornece dados em falta
- utilizador confirma
- utilizador cancela
- canal sem permissão para mutações

### Mensagem principal do diagrama

O bot não executa tudo de imediato; usa um fluxo controlado de proposta -> refinamento -> confirmação -> execução.
